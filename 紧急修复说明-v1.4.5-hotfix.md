# MCTier v1.4.5 紧急修复说明

## 🐛 问题描述

用户反馈：MCTier v1.4.5版本与其他玩家组网后，**无法ping通其他玩家的虚拟IP**。

## 🔍 问题原因

经过排查，发现问题出在EasyTier的启动参数配置上：

**缺少了关键参数 `--ipv4`**

在v1.4.5版本中，启动EasyTier时只配置了`--dhcp true`，但没有指定虚拟网络的IP段（`--ipv4`参数）。这导致：

1. EasyTier虽然能成功启动
2. 虚拟网卡也能创建
3. 但虚拟网卡没有正确配置IP地址
4. 导致无法与其他节点通信（ping不通）

## ✅ 修复方案

在`network_service.rs`的EasyTier启动命令中，添加了`--ipv4`参数：

```rust
cmd.arg("--dhcp")
    .arg("true") // 使用 DHCP 自动分配 IP
    .arg("--ipv4")
    .arg("10.144.144.0/24") // 指定虚拟网络的IP段
    .arg("--hostname")
    .arg(&sanitized_hostname)
    // ... 其他参数
```

### 修复详情

- **虚拟IP网段**：10.144.144.0/24
- **可用IP范围**：10.144.144.1 - 10.144.144.254
- **DHCP模式**：自动分配IP，无需手动配置
- **兼容性**：与之前版本完全兼容

## 🔧 故障排查

如果修复后仍然无法ping通，请检查：

### 1. 防火墙设置
```cmd
# 临时关闭Windows防火墙测试
netsh advfirewall set allprofiles state off

# 测试完成后记得重新开启
netsh advfirewall set allprofiles state on
```

### 2. 虚拟网卡状态
```cmd
# 查看网卡是否启用
netsh interface show interface

# 如果MCTier_Net被禁用，启用它
netsh interface set interface "MCTier_Net" enabled
```

### 3. IP地址配置
```cmd
# 查看虚拟网卡IP
ipconfig /all | findstr /C:"MCTier" /C:"10.144.144"

# 如果没有IP，尝试重启MCTier
```

### 4. 路由配置
```cmd
# 查看10.144.144.0网段的路由
route print | findstr "10.144.144"

# 应该看到类似这样的路由：
# 10.144.144.0    255.255.255.0    On-link    10.144.144.x    281
```
